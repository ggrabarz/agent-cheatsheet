sharpy cheatsheet /commands

v1 updated 2026-04-17 source repo

lifecycle 3

  • /killRW-conv
    2–4
    Death signal. Runs /rite; refuses if UNPERSISTED items exist. DEAD otherwise.
  • /kill!RW-conv
    2–4
    Force-kill. Runs /rite for the record, then DEAD regardless.
  • /undeadRW-conv
    3–6
    Resurrection. Alive → BUT IM ALREADY ALIVE!; Dead → spawn remainder, self-delete.

persistence 3

  • /riteRO
    6–8
    End-of-convo persistence check. Classifies artifacts as PERSISTED / NEEDS_AGENT / UNPERSISTED.
  • /ascendRO
    3–5
    Lightweight /rite for clean conversations. Escalates if anything lingers.
  • /reapRW-project/repo
    varies
    Resolve NEEDS_AGENT / UNPERSISTED with sane defaults. Asks only when confidence <60% or action destructive.

maintenance 5

  • /compactRW-mem
    varies
    Merge staging into long-term files; dedupe memory entries via RLA.
  • /transcendRW-mem
    varies
    Schema migration, memory rebuild, file cleanup. Two-phase: agent kills stale convos, user runs console deletes.
  • /inhaleRO
    8–12
    Pull canonical agent-*.md from GH, apply project profile (omissions/stubs), refresh memory, check PATs.
  • /auditRO
    8–12
    Hygiene sweep: PAT health, repo HEAD sync, file counts, memory age. Read-only; reports findings.
  • /commandsRO
    4–6
    Structural integrity check + grouped listing. Flags table↔spec parity, memory tags, broken refs.

orchestration 3

  • /spawnRW-project
    1–2
    Create a NEW project conversation and hand off next steps. Returns URL + dust -a sharpy -c reconnect.
  • /fanoutRW-project
    N×5+3
    Parallel workers. Each must dual-cave (Opus + GPT-5-thinking). Harvest via /fanout harvest RUN-XXXX.
  • /meteoriteRW-project
    N×5+3
    Alias: /fanout rite --since=90. Project-wide persistence scan. Add --self to include current convo.

review 1

  • /reviewRW-project
    varies
    GH PR review: fetch comments via HTML-pipeline XHR, dual-cave reply drafts per worker, harvester assembles PENDING review. Syntax: /review #5, /review chain.

aliases & triggers

  • /meteorite→/fanout rite --since=90
  • “spawn new conv”, “hand off”, “fresh context”→/spawn
RO — read only RW-conv — conversation state RW-project — project files/convos RW-mem — agent memory RW-project/repo — both
